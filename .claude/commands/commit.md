# Git Commit

변경사항을 확인하고 커밋합니다.

## 커밋 메시지 컨벤션

이 프로젝트는 다음 형식을 따릅니다:

```
<type>: <한국어 설명>
```

### Type 종류
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 변경
- `refactor`: 리팩토링
- `chore`: 빌드, 설정 등 기타 작업
- `style`: 코드 스타일 변경 (포맷팅 등)
- `test`: 테스트 추가/수정

### 예시
```
feat: 친구 페르소나 대화를 무료 사용자에게 개방
fix: 심리케어 API 응답 필드명 불일치 수정
docs: 핵심 가치 및 마케팅 전략 문서 보강
refactor: 페르소나 설정을 커스터마이즈 모달에 통합
```

## 실행할 작업

1. `git status`로 변경사항 확인
2. `git diff`로 상세 변경 내용 확인
3. 변경사항 분석 후 적절한 커밋 메시지 작성
4. 커밋 실행

## 주의사항

- 커밋 메시지는 한국어로 작성
- 변경 내용을 간결하게 요약
- Co-Authored-By 태그 포함
